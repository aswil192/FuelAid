# BIBLIOGRAPHY

1. **AAA.com (American Automobile Association)** - Referred for roadside assistance workflow concepts, service categorization, and emergency response structural designs.

2. **Google Maps / Mapbox** - Used for insights on navigation visualization, filtering service providers by proximity, and structured route planning.

3. **Uber.com** - Referred for real-time service tracking visualization, provider status dashboard layouts, and live progress implementation.

4. **Fuelster.co** - Used to study mobile fuel delivery features, activity feeds for service requests, and engagement mechanics within a roadside assistance context.

5. **NHTSA.gov (National Highway Traffic Safety Administration)** - Referred for vehicle safety information, emergency protocols, and roadside hazard guidelines.

6. **Dribbble.com** - Used as a design reference for modern user interface layouts, bento-grid dashboard styles, and map-centric component styling ideas.

7. **OpenAI.com (ChatGPT models)** - Researched for understanding conversational interaction patterns, natural language location queries, and designing the FuelAid Assistant interface.

8. **Antigravity AI** - Referred for studying advanced agentic AI capabilities, context-aware coding assistants, and modern AI-driven software development practices.

9. **OpenCageData.com** - Used as a primary research source for geocoding metrics, coordinate-to-address conversion, and verified location data guidelines.

10. **Node.js / Express.js** - Referred for researching high-performance RESTful API backend architecture, asynchronous request handling, and efficient database routing implementations.
